# Contributing to AWS EC2 Provisioning with Terraform

First off, thank you for considering contributing to this project 

This project was created to help beginners learn Infrastructure as Code (IaC) using Terraform and AWS. Contributions that improve documentation, code quality, learning resources, and best practices are welcome.

---

## Ways to Contribute

You can contribute by:

- Improving documentation
- Fixing typos or formatting issues
- Enhancing Terraform configurations
- Adding new AWS resources
- Improving project structure
- Suggesting DevOps best practices
- Sharing beginner-friendly explanations

---

## Getting Started

### 1. Fork the Repository

Click the **Fork** button at the top-right corner of the repository.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/terraform-aws-ec2-project.git
````

---

### 3. Navigate into the Project

```bash
cd terraform-aws-ec2-project
```

---

### 4. Create a New Branch

```bash
git checkout -b feature-name
```

---

### 5. Make Your Changes

Update the code, documentation, or infrastructure configuration.

---

### 6. Commit Your Changes

```bash
git add .
git commit -m "Added new feature"
```

---

### 7. Push Your Changes

```bash
git push origin feature-name
```

---

### 8. Open a Pull Request

Submit a Pull Request (PR) describing your changes and improvements.

---

## Contribution Guidelines

Please ensure that:

* Your code is clean and readable
* Terraform configurations are properly formatted
* Documentation is beginner-friendly
* Changes are tested before submission
* Commit messages are clear and descriptive

---

## Terraform Best Practices

When contributing Terraform code:

* Use proper resource naming
* Keep configurations modular when possible
* Avoid hardcoding secrets or credentials
* Use comments where necessary
* Follow Infrastructure as Code (IaC) best practices

---

## Code of Conduct

Be respectful and supportive to all contributors.

This project is intended to be a beginner-friendly learning environment where everyone can learn and grow together.

---

## Questions or Suggestions?

Feel free to open an Issue or start a Discussion.

Thank you for contributing 💪
